# 世足資料來源優先序

分析 FIFA 世界盃時，優先使用最新且可驗證的來源。若來源衝突，先寫出衝突，再採用較權威或較接近即時狀態的資料。

## 賽程、場地、賽制與出線情境

1. FIFA 官方賽程、比賽中心、賽事規章、紀律公告與積分表。
2. 主辦單位、球場官方公告與賽事操作資訊。
3. 各國足協與國家隊官方公告。
4. FotMob / SofaScore / ESPN / BBC 等賽程頁。
5. 其他比分網站。

官方來源對開賽時間、官方主客、場地、延期、積分排序、停賽、正式先發與淘汰賽結算規則優先。小組第三輪必須確認同組另一場是否同步開球，以及即時積分與淨勝球情境。

## 正式先發與預計先發

1. FIFA 官方比賽中心或國家隊官方先發公告。
2. 各國足協社群、主教練賽前記者會與可靠隊記。
3. FotMob / SofaScore / WhoScored 的已確認先發。
4. FotMob / SofaScore / WhoScored 的預計先發。
5. 最近國家隊出賽、訓練分組、球員俱樂部角色與輪換規律推估。

正式先發未公布前，只能寫 `預計` 或 `推估`。賽前約 60 分鐘若正式先發公布，必須優先更新。

## 傷停、停賽與可用性

1. 國家隊官方公告、主教練記者會、官方傷病更新。
2. FIFA 紀律報告、紅黃牌累積與停賽公告。
3. 可靠隊記、當地權威媒體與球員俱樂部醫療/出場資訊。
4. FotMob / SofaScore / WhoScored 傷停頁。
5. Transfermarkt 傷停/停賽列表。

若球員被列為出賽成疑、狀態待確認、需賽前體能測試或未完整訓練，不要當成確定出賽。國家隊賽事要特別檢查俱樂部近期出場時間、傷後復出節奏與是否被限制上場時間。

## 進階數據

- FIFA 官方比賽資料與技術報告：控球、射門、角球、犯規、牌數、站位與賽後報告。
- FIFA 官方技術報告、Opta／The Analyst 報導：預期進球、非點球預期進球、預期失球、射門品質與戰術資料。
- FotMob／SofaScore／WhoScored：比賽事件、供應商標示的預期進球、平均站位、預計陣型、近期狀態與即時數據；不同供應商的 xG 不直接混算。
- FBref：只使用仍可取得的基礎與歷史資料。自 2026 年 1 月進階資料被移除後，不再把 FBref 當作 StatsBomb／進階 xG 來源。
- 球員俱樂部資料：近期上場時間、位置角色、傷病恢復與狀態，只作國家隊模型輔助。

近幾場勝敗、控球率與場均進球只能作背景，不能取代預期進球、射門品質、對手強度、先發脈絡與賽制情境。資格賽對弱隊刷出的進攻數據要下修。

## 戰術與近期內容

1. 最近 3 到 5 場國家隊完整比賽或官方長版精華。
2. 主教練記者會、官方訪談與訓練報導。
3. 可靠戰術分析、賽後報告與當地媒體。
4. 數據網站的陣型、平均站位、射門位置圖與球員熱區。

若無法看完整比賽，必須說明限制，並提高可驗證數據與近期報導的權重。

## 天氣、場地、裁判與主辦因素

1. FIFA 官方場地與賽事公告。
2. 球場資訊、草皮狀態、屋頂、海拔與城市移動距離。
3. 可靠天氣來源：溫度、濕度、風、雨與悶熱程度。
4. 裁判指派與歷史牌數/點球傾向。
5. 主辦國、鄰近國家或大僑民球迷造成的實際主場感。

裁判與天氣只能作輔助，不可過度放大；但極端風雨、悶熱、高海拔、差草皮或嚴格裁判可影響大小球、角球與牌數市場。

## 盤口

1. 使用者提供的即時賠率。
2. 使用者指定的盤口來源。
3. 多市場平均價格作參考。

分析時以模型機率為主，再比較市場價格。不要把賠率變動直接解讀為必然資訊優勢。世界盃市場結算規則差異很大，必須先確認 90 分鐘、含延長賽、晉級或 PK。

報告輸出的賠率只允許十進位。若來源提供美式、分數、香港盤、馬來盤、印尼盤或其他格式，先內部轉換；無法可靠轉換時，不輸出原始賠率，只保留模型機率、公允十進位賠率與價格門檻。

## 賽後檢討資料

1. FIFA 官方比賽中心：正式比分、進球、牌、換人、先發與賽事報告。
2. FotMob / SofaScore / WhoScored：預期進球、射門品質、重大得分機會、平均站位、球員評分與比賽事件。
3. FIFA 技術資料、Opta／The Analyst 或明確標示供應商的數據頁：用於預期進球、射門位置圖、非點球預期進球與攻防內容回測；不同 xG 模型不可無註記混用。
4. 可靠賽後報導與教練/球員訪談：用於確認傷病、戰術調整與臨場計畫。
5. 盤口紀錄或使用者提供截圖：用於檢查原推薦是否有真實期望值，以及收盤是否反向。

檢討時必須區分結果運氣與流程錯誤。若原方向贏預期進球卻輸比分，可標記為合理變異；若比分、預期進球、射門品質與戰術內容都反向，必須視為模型錯誤並降溫。

## 來源衝突處理

- 賽程與場地：FIFA 官方優先；若比分網站時間不同，以官方或倒數時間更新較近者為準並標註。
- 先發：正式先發優先於預計先發；未公布時以預計先發與輪換規律推估並降信心。
- 傷停：官方與可靠隊記優先；資料互相衝突時，列出狀態差異並避免高信心下注。
- 停賽：FIFA 紀律公告優先。
- 出線情境：FIFA 積分表與賽事規章優先；自行計算時要列明積分、淨勝球、進球數與排序規則。
- 數據：若預期進球與比分近況衝突，解釋差異後優先使用能反映機會品質的指標。
- 盤口：記錄查詢時間點。盤口快速變動時，建議使用者提供即時價格再判斷期望值。
