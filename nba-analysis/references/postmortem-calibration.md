# NBA 賽後檢討與模型校準

## 1. 重建賽前快照

- 核對資料截止時間、官方 Injury Report 版本、Questionable／GTD、正式先發、minutes restriction 與 late scratch。
- 核對主客場、休息、背靠背、旅行、系列賽比分與原盤口結算線。
- 保存原獨贏、讓分、大小分、隊伍總分、信心度與推薦，不用賽後資訊改寫。

## 2. 重建比賽

- 實際先發、輪替與上場時間；缺陣或受限球員是否落在原情境內。
- 回合數、進攻／防守效率、Four Factors、shot profile、罰球、籃板、失誤與垃圾時間。
- 比較上半場與下半場、替補段、犯規麻煩、教練調整與 blowout 發生時間。
- 來源以 NBA 官方 box score／play-by-play、官方傷兵報告與可追溯 lineup data 為主。

## 3. 評分

- 獨贏使用 Brier score；讓分與大小分使用原結算線的 cover／push／no-cover 分布評分。
- 同時記錄預估分差、總分與實際殘差，但把垃圾時間與 OT 另行標記。
- 以例行賽／季後賽、pre-lineup／post-lineup、傷兵狀態與信心區間分組校準；不以一場三分爆發改整體權重。

## 4. 錯誤歸因

- 資料錯漏：傷兵報告版本、先發、輪休、minutes restriction 或 late scratch。
- 情境權重：Questionable 球員出賽／缺陣機率、替代持球點或輪替分鐘估錯。
- 對位模型：pace、shot profile、護框、籃板、犯規或替補段誤判。
- 相依分布：獨贏、讓分與大小分是否沒有從同一分差 × 總分分布推導。
- 合理變異：三分、罰球、clutch、OT；若流程與機會品質也反向，視為模型錯誤。

## 5. 修正與輸出

- 流程錯誤立即修正；單場賽果不硬設下一場勝率或信心上限。
- 若正式先發改變主分布，建立 post-lineup 新快照，保留 pre-lineup 版本供檢討。
- 推薦失準時先降一級注碼；模型信心度仍依五項組成重算。
- 輸出依序列出：賽果與來源、原預測 vs 實際、情境命中、機率評分、錯誤歸因、模型修正。
