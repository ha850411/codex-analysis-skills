# MLB 輸出模板

完整分析使用此模板。若使用者要求簡版，可保留核心欄位並壓縮文字。

先標示模型層級：`validated-production`、`public-data-baseline` 或 `unmodeled`。沒有生產模型時先執行公開資料 baseline；只有 baseline 也失敗時才交付 `degraded` 本地報告並填 `N/A`。自動排程不得把全 N/A 報告發布成正常每日預測。

## daily-summary 壓縮規則

整日多場報告使用下列順序，不把後面的完整單場模板複製 10–17 次：

1. 全日結論：模型層級、數值覆蓋率、最高／最低方向性勝率、觀測 Tier 分布、逐場模型信心度範圍、下注資格與資料截止。不得把 baseline 驗證狀態或舊版上限當成全日固定信心度。
2. 今日賽程盤點：全部場次一張表。
3. 全場模型比較：全部場次一張表，至少列雙方期望得分、總分 80% 區間、方向性全場勝率與公允賠率、明確隊名前五局三路、基準大小分與勝率、模型信心度與場次專屬風險。
4. 觀察名單：最多展開 3 場，說明先發、打線／牛棚／環境缺口、進場門檻價格與翻轉條件；baseline 標註 0u 但給出條件門檻。
5. 方法、來源、共通風險與免責。
6. 全文最後輸出唯一的 `簡表總結`。

### 欄位格式規範（全場比較表與簡表總結必須遵循）

- **全場勝率（含方向性與公允價）**：必須完整標示雙方勝率、勝方傾向與公允賠率。格式：`[勝方傾向] [勝隊] [勝率]% (公允 [賠率]) | [被看淡隊] [勝率]%` (例：`主勝傾向：水手 54.92% (公允 1.821) | 紅人 45.08%`)。
- **前五局三路**：必須明確標示客隊、平手與主隊。格式：`[客隊簡稱] [客勝]% / 平手 [平]% / [主隊簡稱] [主勝]%` (例：`紅人 36.58% / 平 16.95% / 水手 46.47%`)。
- **大小分預測**：必須包含模型預估總分、80%區間與基準盤口傾向（Over/Under勝率）。格式：`[預估得分]分 (80%區間 [p10]–[p90]) | [大/小分傾向] (盤口 [線]分, [勝率]%)` (例：`8.40 分 (3–14) | 小分傾向 (< 8.5 分, 52.4%)`)。
- **推薦 / 觀測與進場門檻**：`public-data-baseline` 雖然注碼固定 0u，但必須給出方向標的、公允價格、進場門檻與觀測 Tier。格式：`[觀測 Tier] [標的] (公允 [賠率], 進場門檻 >= [門檻賠率]) | 0u` (例：`[Tier 1 觀測首選] 水手 ML (公允 1.82, 門檻 >= 1.85) | 0u`)。
- **核心風險**：必須列出**場次專屬實務風險** (如：`雙重賽 G2 晚場投手與打線異動`、`BOS 先發 TBD`、`Coors Field 高海拔極端總分波動`、`雙方接近 50/50 波動高`)，嚴禁逐場複製相同的「需另建快照 / 未納入牛棚」全域樣板文。
