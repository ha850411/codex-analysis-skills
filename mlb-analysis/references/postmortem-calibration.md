# MLB 賽後檢討與模型校準

## 1. 重建賽前快照

- 核對資料截止時間、probable／正式先發、球數或局數限制、正式打線、牛棚近三日用量、天氣與屋頂狀態。
- 分開保存前五局與全場的原機率、平手／走盤機率、信心度、盤口與推薦。
- 雙重賽、延賽、opener、bullpen game 或臨時換投要建立獨立快照。

## 2. 重建比賽

- 先發投手球種、球速、揮空、保送、被打品質、Times Through Order 與實際退場原因。
- 正式打線、左右投對位、關鍵休兵、守備／跑壘與捕手配置。
- 牛棚使用順序、勝利組可用性、長中繼、連投與高 leverage 決策。
- 球場、風、溫度、雨、屋頂與主審資訊只在當時可知且具影響時納入。

## 3. 評分

- 全場獨贏與前五局含平手分布使用 Brier score；需要比較模型時另記 log loss。
- 記錄前五局與全場得分殘差，分開檢查先發模型與牛棚模型。
- 以 pre-lineup／post-lineup、投手狀態、球場類型與信心區間建立 calibration cohort；單場全壘打或守備失誤不直接改權重。

## 4. 錯誤歸因

- 資料錯漏：先發更換、球數限制、打線、牛棚不可用、天氣或屋頂。
- 投手模型：pitch mix、球速、K-BB%、被打品質、左右拆分或 TTO 判錯。
- 打線模型：小樣本火力、對手強度、platoon、休兵或傷後狀態誤判。
- 牛棚模型：預期接手機制、連投、closer／setup 可用性或延長賽深度誤判。
- 相依分布：前五局與全場、獨贏與大小分是否被分別手填。
- 合理變異：BABIP、守備、單次全壘打、滿壘殘壘或裁判；若被打品質與流程也反向，視為模型錯誤。

## 5. 修正與輸出

- 流程錯誤立即修正；單場結果不硬設下一場機率上限。
- 前五局正確但全場錯時，優先檢查牛棚情境，不推翻先發投手模型。
- 正式打線或天氣改變後建立新版快照，保留原版本供校準。
- 輸出依序列出：賽果與來源、原預測 vs 實際、前五局／全場拆解、機率評分、錯誤歸因、模型修正。
