# 機率建模與校準契約

所有產生新機率的 `*-analysis` skill 都套用本契約。領域規則負責選擇變數與結算方式；本文件負責預測快照、樣本處理、分布一致性、信心度與賽後校準。

## 1. 鎖定預測快照

每次預測先記錄 `as_of`、賽事身分、賽制、已知名單／先發、版本／地圖池／場地與資料狀態。以下任一資訊出現後，不得悄悄覆寫原機率；要建立新版快照並說明變更原因：

- 正式先發、stand-in、傷停或投手／球員可用性改變。
- 已公布 veto、draft、選邊、地圖順序、屋頂／天氣或賽事規則改變。
- 比賽時間、對手、賽制或市場結算範圍改變。

`pre-lineup`、`post-lineup`、`pre-veto`、`post-veto`、`pre-draft`、`post-draft` 是不同資訊集。只比較同一快照的機率與價格；賽後檢討也要使用當時可知的資訊。

## 2. 證據與樣本處理

1. 先建立基準強度，再用近期、對手、場地與當前版本修正；不得只用最近幾場勝敗取代基準率。
2. 「近 14 天」「近 30 天」「賽季」若互相包含，不可直接相加成獨立權重。使用不重疊區間、逐場時間衰減或明確去重後再彙整。
3. 對小樣本做收縮，不直接採用原始勝率。可用下式或等價的階層／貝葉斯方法：

   ```text
   收縮後估計 = (有效樣本數 × 樣本估計 + 先驗強度 × 基準率)
                / (有效樣本數 + 先驗強度)
   ```

4. 有效樣本數要因跨版本、跨陣容、對手強度不相當、垃圾時間、友誼賽或低層級賽事而折價。
5. 同一事實衍生出的多個指標只算一組證據。例如近期勝率、淨勝分與排名變化若來自同一批比賽，不可各自滿額加權。
6. 缺資料時保留先驗並擴大不確定性；不得用敘事補出不存在的數字。

領域 skill 內的百分比權重是起始配置，不是每場強制常數。調整時需說明哪個資料區塊因樣本、時效或制度變動而升降權。

## 3. 情境集成

對未確定資訊建立少量可解釋情境，不用單一猜測冒充確定事實：

```text
最終機率 = Σ 情境機率 × 該情境下的結果機率
```

- 情境權重本身需有來源或明確理由。
- 低機率但高影響的傷停、替補、veto、draft、天氣或輪休要進入風險，不可完全刪除。
- 正式資訊公布後建立新版快照，移除已不可能的情境並重算。

## 4. 先建主分布，再推導市場

不可分別憑直覺填入勝率、精確比分、至少一局、大小分與讓分。先建立一個互斥且完備的主分布，再從同一分布確定性推導所有相依機率：

- LoL／CS／Valorant／Dota 2：以各局／各圖在可能 veto、選邊或 draft 情境下的勝率建立精確系列比分分布；系列賽勝率、至少一局／一圖、橫掃與總局數都由精確比分相加得到。不得假設每局獨立同分布；局間調整與地圖／英雄池差異要進入情境。
- 足球：先建立 90 分鐘正確比分矩陣，再推導三路盤、讓球、大小球、雙方進球與各隊進球數。淘汰賽的延長賽／PK／晉級另建條件分布，不能混入 90 分鐘矩陣。
- NBA：先建立得分差與總分的聯合分布，或以回合數與每回合得分情境模擬；再推導獨贏、讓分、大小分、隊伍總分與半場市場。
- MLB：分開建立前五局與全場得分分布；全場分布要加入先發退場後的牛棚情境，再推導獨贏、讓分、大小分與隊伍總分。

若無法建立足以支撐某衍生市場的主分布，該市場填 `N/A（原因）`，不要獨立造數字。

## 5. 機率解析度與不確定性

- 預設輸出整數百分比；有實際模擬或可重現計算時最多保留一位小數。
- 不用 0.01% 等假精準度。四捨五入後，互斥完備分布仍須在 `100 ± 0.2` 內。
- 主要機率接近五五開、對未確認情境敏感或不同合理模型差異大時，除了點估計，補充敏感區間或翻轉條件。
- 「最可能結果」不等於高信心；結果分布的峰值與資料可靠度分開表達。

## 6. 模型信心度

模型信心度使用可重現的五項評分，不使用市場價格、賠率差或投注價值：

| 組成 | 權重 | 判斷重點 |
| --- | ---: | --- |
| 資料完整度 | 25% | 必要欄位、來源與關鍵變數是否齊全 |
| 資料新鮮度 | 20% | 擷取時間是否符合該資訊的變動速度 |
| 名單／先發確定度 | 25% | roster、傷停、先發、投手、veto／draft 是否已確認 |
| 制度與樣本相關性 | 20% | 同版本、同陣容、同賽制、同層級對手的有效樣本 |
| 模型穩定性 | 10% | 合理情境或方法變動後，主要機率是否穩定 |

```text
模型信心度 = 0.25×完整度 + 0.20×新鮮度 + 0.25×先發確定度
             + 0.20×樣本相關性 + 0.10×模型穩定性
```

四捨五入為整數百分比，並在內部保留五項分數。若領域確實不適用「先發」，以等價的賽前可用性資訊取代，但不得把該權重移給市場。任何額外信心上限都必須指出觸發的資料或流程缺口。

## 7. 市場比較

先鎖定模型機率與信心度，再讀市場。市場分歧可以觸發資料遺漏檢查，但不能直接改機率；若找到新的賽事事實，建立新版快照後從主分布重算。

下注建議使用不確定性調整後的機率，不把點估計的全部優勢當成可實現優勢。價格過期、結算不明或關鍵先發未定時，只給價格門檻或觀望。含走盤、半贏、半輸的市場必須用正確結算期望值，不可把走盤當輸。

## 8. 賽後評估

所有正式檢討先套用 `postmortem-improvement.md` 的改善閉環與 PR 證據契約。

每筆有結果的預測至少保留：快照時間、點估計、實際結果、模型信心度、資料缺口與主要情境。使用以下原則評估：

- 二元或多類機率使用 Brier score；需要時另記 log loss。不可只用「有沒有猜中」評斷機率品質。
- 以運動、賽制、預測時點與信心區間分組做 calibration table；樣本不足時只標記，不宣稱已校準。
- 單場冷門不自動改權重或永久壓低某隊機率。只有找到可重複的流程錯誤時才立即修正；純結果變異要累積同類樣本後再調整。
- 若是資料錯漏、結算混用、市場洩漏或相依機率不一致，視為流程錯誤，立即修正且不等待更多樣本。
- 比較舊、新流程時使用同一批歷史快照做 walk-forward／時間序列回測，不讓賽後資訊滲入賽前特徵。
- 只降低信心度、注碼、推薦等級或輸出語氣不算模型改善；沒有基準版與挑戰版的同批驗證，不得宣稱精準度已提升。
