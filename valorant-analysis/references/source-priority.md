# Valorant 資料來源優先序

## 賽程、賽制、Patch 與 Map Pool

1. Valorant Esports、Riot／賽區官方賽事頁、主辦方公告與 rulebook。
2. VLR.gg Match Page 與 event page。
3. Liquipedia Valorant tournament page。

官方來源對改期、賽制、Patch、Map Pool、特殊 veto 規則與當場裁定優先。VLR 的倒數與 Match Page 適合交叉查核；若與官方衝突，採時間較新的官方更新並揭露差異。

## 當場 roster、stand-in 與選手可用性

1. 主辦方當場 roster、隊伍公告或聯賽官方公告。
2. VLR Match Page 的 match roster／projected lineup。
3. Liquipedia roster。
4. 最近一場正式賽出賽五人。

Projected lineup 不是正式先發。替補、簽證、健康或角色變動未確認時建立情境，並降低名單／先發確定度。

## 地圖、特務與選手數據

1. Valorant Esports match centre、官方 box score／VOD。
2. VLR match history 與地圖紀錄。
3. RIB.gg、THESPIKE 或其他可追溯的回合／選手數據。
4. Liquipedia 作賽事脈絡補充。

資料必須標記 Patch、陣容、對手層級、地圖樣本數與擷取時間。跨 Patch、跨 Map Pool 或跨 roster 的原始勝率先收縮，不直接沿用。

## 已公布 veto 與賽後資料

- 官方 broadcast／match centre 公布的 veto、map order、pick owner 與選邊優先。
- VLR／Liquipedia 只作交叉查核；若資訊仍不一致，標記未確認並降低信心。
- 賽後以官方結果、每圖比分、VOD 與可追溯數據重建，不用社群敘事替代關鍵回合。

## 盤口

使用者提供的即時賠率優先，其次為使用者指定來源或可記錄擷取時間的多市場價格。所有賠率轉為十進位，只在模型快照鎖定後用於價格與期望值比較，不得回寫逐圖或系列賽機率。
