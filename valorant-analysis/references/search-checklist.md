# Valorant 搜尋檢核表

每次分析前檢查：

1. 日期是否已換算成台灣時間 UTC+8？
2. 是否已搜尋官方賽事頁與 VLR 當日賽程？
3. 是否已用官方更新、VLR 與 Liquipedia 交叉查核，並按 `source-priority.md` 處理衝突？
4. 是否已列出所有找到的當日比賽，而不是只分析一場？
5. 每場 BO1 / BO3 / BO5 是否確認？
6. 若只找到少量場次，是否已用賽事名稱再搜一次？
7. 是否確認先發 5 人、主 Duelist、Controller、IGL/Caller、stand-in？
8. 是否確認近期 Patch / Map Pool 是否影響舊數據？
9. 若核心特務或主 Duelist 受 Patch 影響，是否下修舊資料並限制信心度？
10. 是否檢查雙方同 Patch / 同賽事樣本，避免低估熱手隊伍？
11. 最近 H2H 是否跨 Patch、跨賽制或跨地圖池，是否避免過度錨定？
12. 若排名或數據站與模型方向相反，是否重新檢查具體賽事證據，並確認沒有把市場賠率當成反證或信心度修正依據？
13. 是否確認當前賽事可用 Map Pool，並逐張分析兩隊每一張地圖？
14. 逐圖分析是否包含兩隊樣本、攻守表現、常用特務陣容、對位重點、單圖傾向與 Veto 意義？
15. 若官方 Map Veto 已公布，是否改用 post-veto 重算，而不是沿用賽前 veto 預測？
16. 是否檢查 pick owner、選邊、left-through signal 與對手 anti-strat 可能性？
17. 是否完成 Map Veto 推演，並檢查 BO3 的 2-0 / 反掃風險？
18. BO5 是否列出 3:0、3:1、3:2 與反向 3:2 路徑，避免把三張小優圖硬寫成穩定 3:1？
19. 落敗方至少一圖若高於 75%，是否有一張明確優勢圖或兩條獨立取圖路徑支持？
20. BO1 是否避免輸出 +1.5 maps？
21. 若引用賠率或市場價格，是否只用十進位輸出並避免保留原始非十進位格式？
22. 若使用者要求賽後檢討、失準歸因或模型校準，是否先讀 `references/postmortem-calibration.md`？
23. 是否記錄 `as_of` 與 pre/post-veto 快照，避免公布 veto 後覆寫原預測？
24. 獨贏、精確比分、至少一圖、橫掃與總地圖數是否都由同一精確比分主分布推導？
25. 模型信心度是否依五項組成計算，且完全排除市場價格？
